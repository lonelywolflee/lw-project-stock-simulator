현재 스테이징되지 않은 변경사항을 분석하고, 단일 목적별로 분리하여 커밋해주세요.

## 규칙

1. **커밋 타입 분류**: 모든 커밋 메시지는 반드시 다음 4가지 접두사 중 하나로 시작해야 합니다:
   - `docs:` — 문서 변경 (README, PRD, 주석 등)
   - `feat:` — 새로운 기능 추가
   - `refactor:` — 기능 변경 없는 코드 구조 개선
   - `fix:` — 버그 수정

2. **단일 목적 커밋**: 하나의 커밋에는 하나의 목적만 포함해야 합니다.
   - 서로 다른 목적의 변경사항이 섞여 있으면 **별도의 커밋으로 분리**하세요.
   - 예: 새 기능 추가와 기존 버그 수정이 함께 있으면 `fix:` 커밋과 `feat:` 커밋으로 나눠야 합니다.

3. **커밋 메시지 형식**:
   ```
   <type>: <간결한 설명 (한국어)>
   ```
   - 설명은 한국어로 작성하되, 코드 식별자는 원문 유지
   - 50자 이내로 간결하게 작성

## 절차

1. `git status`와 `git diff`로 모든 변경사항을 파악합니다.
2. 변경사항을 목적별로 그룹핑합니다.
3. 각 그룹을 해당 파일만 `git add`하여 개별 커밋합니다.
4. 커밋 완료 후 `git log --oneline -10`으로 결과를 보여줍니다.
